## Purpose

A collection of **reusable GitHub Actions and reusable workflows** consumed by other repositories in the MockaSort-Studio organisation. No application code lives here — only CI/CD primitives.

## Tooling conventions

- **Python packaging**: `uv` exclusively — never `pip` or `poetry`
- **Setup**: `astral-sh/setup-uv@v5` with `enable-cache: true`
- **Python version**: declared via `.python-version`; pass through `python-version` input when forwarding
- **Testing**: `pytest` via `uv run pytest`

## Composite actions — key contracts

| Action | Purpose | Notable inputs |
|--------|---------|----------------|
| `test-uv` | Run pytest in-tree | `python-version`, `test-args` |
| `smoke-test-uv` | Install from wheel+sdist and run pytest | `python-version`, `artifact-name`, `enable-cache` |
| `generate-coverage` | Build + test + emit coverage badge | `python-version`, `package-name`, `working-directory` |
| `lint-check` | Run linters | — |
| `changes-filter` | Detect changed paths | `match-paths.py` driven |
| `build-bazel` | Bazel build + test via bazelbuild/setup-bazelisk@v3 | `targets`, `config` (optional, omitted if empty) |
| `push-container` | Build and push Docker image | — |

## Hard constraints

- Never use `pip` — `uv` only
- Do not modify `.github/workflows/actions-checks.yml` without understanding the PR-comment trigger pattern
- CI checks triggered by posting `:pig_nose:` comment on a PR
- Composite actions use `runs: using: composite` — not Docker, not JS
- Reusable workflows must declare their own `permissions` (callers should not need to specify them)
